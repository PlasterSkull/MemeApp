# CLAUDE.md Restructure Design

**Date:** 2026-05-16  
**Status:** Approved

## Summary

Split single `CLAUDE.md` into a navigable knowledge base under `.claude/knowledge/`. Root `CLAUDE.md` becomes project overview + table of contents. Each category gets its own `README.md` index and individual topic files. Docs load on-demand per task, not all at once.

## Goals

- Reduce irrelevant context per task (load only what's needed)
- Separate business knowledge from technical docs from code style rules
- Easy navigation: root TOC → category README → specific file
- Keep `.claude/` out of project docs; this is Claude-tooling, not project documentation

## File Structure

```
CLAUDE.md                              ← project overview + TOC to knowledge base
.claude/
  knowledge/
    project/
      README.md                        ← TOC for project knowledge
      overview.md                      ← business description, goals, scope
      domain.md                        ← domain entities: Meme, Tag, Collection, MediaType
    architecture/
      README.md                        ← TOC for architecture docs
      overview.md                      ← layers, bounded contexts, solution structure
      fusion.md                        ← ActualLab.Fusion architecture patterns
    code-style/
      README.md                        ← TOC for code style rules
      naming.md                        ← classes, interfaces, DTOs, methods, variables
      file-namespace.md                ← file-scoped namespaces, file organization
      di-modules.md                    ← DI registration, Configure, AddModules
      fusion.md                        ← Fusion-specific patterns: IComputeService, IComputed
      async.md                         ← async/await, cancellation, ValueTask
      formatting.md                    ← Allman braces, expression bodies, var vs explicit
      blazor-ui.md                     ← Blazor components, Fusion+Blazor integration
      testing.md                       ← test structure, naming, what to mock
```

## Root CLAUDE.md Content

Sections:
1. **Project** — one-paragraph description of MemeApp (meme sorting system: photos, videos, gifs)
2. **Build Commands** — dotnet build/restore/clean
3. **Knowledge Base** — TOC with paths to each category README
4. **Package Management** — centrally managed via Directory.Packages.props

Removed from root: Key Libraries list (moved to architecture/overview.md), Code Style section (moved to code-style/), detailed architecture (moved to architecture/overview.md).

## Navigation Convention

When starting a task:
- Code task → read `.claude/knowledge/code-style/README.md`, then relevant files
- Architecture task → read `.claude/knowledge/architecture/README.md`
- Business/domain task → read `.claude/knowledge/project/README.md`
