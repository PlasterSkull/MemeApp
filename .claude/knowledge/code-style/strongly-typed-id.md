# Strongly Typed IDs

Use `StronglyTypedId` for all domain entity IDs. Prevents mixing IDs of different types at compile time.

## Declaration

```csharp
[StronglyTypedId]
public readonly partial struct MemeId
{
    public bool IsNone => this == Empty;
}

[StronglyTypedId]
public readonly partial struct TagId
{
    public bool IsNone => this == Empty;
}

[StronglyTypedId]
public readonly partial struct CollectionId
{
    public bool IsNone => this == Empty;
}
```

Rules:
- `readonly partial struct` — not class, not plain struct
- `IsNone => this == Empty` — `Empty` is generated by the source generator
- One file per ID type: `MemeId.cs`, `TagId.cs`, etc.

## Where to Use

**Use strongly typed IDs:**
- Domain entity classes (`Meme`, `Tag`, `Collection`)
- Service method parameters and return types

**Do NOT use strongly typed IDs:**
- EF Core entity model classes — use `Guid` there for ORM compatibility
- Database migrations
